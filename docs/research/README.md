# Research — grounding references

Read-only research artifacts that **ground** formatting- and color-fidelity work in the
`tableau-migration` skill. These are repo-level references, **not** part of the shipped skill
payload under `skills/tableau-migration/` — so they are not mirrored to `plugins/` and do not
affect the skill `VERSION`.

## Power BI formatting & color — serialization reference

A corpus-grounded reverse-engineering of how Power BI **serializes and resolves visual formatting
and color**, produced for the Tableau → Power BI dashboard-migration path.

| File | What it is | Use it for |
| --- | --- | --- |
| [`powerbi-formatting-color-reference.md`](./powerbi-formatting-color-reference.md) | Reasoned synthesis (~1,100 lines): value grammar, color forms, the 5-layer precedence model, a per-visual-type matrix, and a Tableau → Power BI mapping table rated fit / partial / GAP. | The human-readable spec + design rationale when building a formatting feature. |
| [`powerbi-formatting-inventory.json`](./powerbi-formatting-inventory.json) | Machine-readable companion (~1.4 MB): a flat `object → property → type → default → layer → Tableau-equivalent → sources` index. | Programmatic lookups / a test oracle. **Not shipped** (size); repo provenance only. |

**Corpus.** 52 legacy `Report/Layout` `.pbix` + 1 enhanced-format (PBIR) `.pbip`, deduplicated by
MD5. Root JSON sections: `objectIndex` (113 objects), `byVisualType` (49 types), `themesObserved`
(76), `coverageMatrix` (features × files), plus `valueGrammar`, `themeSchema`, `precedenceModel`,
and `categoryToTableau`.

**Evidence tags** (used throughout both files — check them before relying on a claim):
`[CORPUS]` observed in the corpus · `[SCHEMA]` from the official report-theme JSON schema ·
`[PBIR]` enhanced-format structure (paths schema-grounded, not value-observed) · `[INFERRED]` our
interpretation, verify before use · `⇄ Tableau` a Tableau-equivalence note (`GAP` = no clean
counterpart).

**What it feeds.** Formatting-fidelity features in the migration skill — e.g. Visual Calculation
`formatString`, rule-based conditional formatting (`values.backColor` / `fontColor`), axis
reference lines, in-cell data bars, and opacity / row-banding.

**Provenance / regeneration.** Generated by a research pass over the public Power BI sample corpus;
the reproduction method is documented inside the reference `.md`. These files are grounding
evidence, not generated code — treat them as read-only.
